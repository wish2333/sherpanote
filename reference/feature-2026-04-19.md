- 本地模型指定方式

  - 下面以通过初始化参数传入为例：

    ```python
    from rapidocr import RapidOCR
    engine = RapidOCR(params={"Det.model_path": "models/ch_PP-OCRv4_det_infer.onnx"} ) 
    img_url = "https://img1.baidu.com/it/u=3619974146,1266987475&fm=253&fmt=auto&app=138&f=JPEG?w=500&h=516"
    result = engine(img_url)
    print(result)
    result.vis("vis_result.jpg") 
    ```

    上面第 4 行通过 `Det.model_path` 指定了本地已经下载好的文本检测模型。文本方向分类和文本识别模型也可同样指定。(`Cls.model_path` 和 `Rec.model_path` 同理)

  - 其他参数设置

    ```
    from rapidocr_onnxruntime import RapidOCR
    
    # 自定义配置
    ocr = RapidOCR(
        det_model_path='models/det_model.onnx',      # 检测模型路径
        rec_model_path='models/rec_model.onnx',      # 识别模型路径
        cls_model_path='models/cls_model.onnx',      # 分类模型路径
        det_thresh=0.5,                               # 检测阈值
        rec_batch_num=6,                              # 识别批处理大小
        use_cuda=False,                               # 是否使用GPU
        engine_type='onnxruntime'                     # 推理引擎类型
    )
    ```

### PLAN

● OCR Feature Implementation Plan for SherpaNote v2.0.0

  Requirements Summary

  Add OCR functionality using RapidOCR > 3.8.0 + OnnxRuntime + local models (PP-OCRv4/v5). The feature includes:
  - New OCR view on main page for image/PDF upload and processing
  - Two processing modes: batch (one record per image) and sequential (combined record)
  - PDF support (pages rendered to images, then processed sequentially)
  - OCR settings section in the settings page
  - Full integration with existing record system

  Implementation Phases

  Phase 1: Backend - OCR Engine Module

  - New file: py/ocr.py (~250 lines) - OcrEngine class wrapping RapidOCR + pdf_to_images() using PyMuPDF
  - Modify: py/config.py - Add OcrConfig frozen dataclass (model_version, det_thresh, rec_batch_num, use_cuda) to
    AppConfig
  - Modify: pyproject.toml - Add PyMuPDF dependency

  Phase 2: Backend - API Endpoints (in main.py)

  - ocr_process(files, mode, title) - Core processing endpoint, runs in background thread, emits progress events
  - pick_image_files() - Native file dialog for images/PDF
  - check_ocr_models() - Scan data/rapidocr/ for model availability
  - get_image_preview(path) - Return base64-encoded image preview
  - cancel_ocr() - Cancel ongoing OCR processing
  - Reset OCR engine on config update

  Phase 3: Frontend - Types and Bridge

  - Modify: types/index.ts - Add OcrConfig, OcrModelStatus, OcrMode, OcrFileEntry interfaces
  - Modify: bridge.ts - Add pickImageFiles(), checkOcrModels(), ocrProcess(), getImagePreview() helpers

  Phase 4: Frontend - OCR View

  - New file: views/OcrView.vue (~400 lines) - Upload area (drag-drop + button), mode selector, file list with previews,
      process button, progress bar
  - Modify: router/index.ts - Add /ocr route
  - Modify: App.vue - Add "OCR识别" nav item + OCR progress event listeners

  Phase 5: Frontend - OCR Settings

  - Modify: SettingsView.vue - Add OCR settings tab (model version selector, threshold slider, batch size, CUDA toggle,
    model status table)
  - Modify: stores/appStore.ts - Add ocrConfig ref + OCR progress state

  Phase 6: Integration and Polish

  - Verify OCR records appear correctly in HomeView (they reuse records table with audio_path=None)
  - Optional: backup module update for OCR models
  - Cleanup temp files from PDF conversion

  Key Design Decisions

  1. OCR records reuse existing records table - audio_path=None, transcript holds OCR text, segments hold per-image text
      blocks
  2. PyMuPDF for PDF conversion - Pure Python, no system dependency (unlike pdf2image + poppler)
  3. Lazy OCR engine init - Same pattern as existing _get_asr(), only loads when first needed

### 修复策划

- 模型下载方式：通过完成一次带参数的实例化完成下载，模型保存目录在：

  - 开发环境：.venv\Lib\site-packages\rapidocr\models
  - 打包环境：这个如何识别你自己想，通过Pyinstaller打包的，会在Win和Mac上打包
  - 带参数，包括真实程序中也改成这样的传参（不再传入CUDA，没有明显加速），参考：

  ```python
  from rapidocr import ModelType, OCRVersion, RapidOCR
  
  self._engine = RapidOCR(
   params={
    "Det.ocr_version": OCRVersion.PPOCRV5,
    "Rec.ocr_version": OCRVersion.PPOCRV5,
    "Cls.ocr_version": OCRVersion.PPOCRV5,
    "Det.model_type": ModelType.MOBILE,
    "Rec.model_type": ModelType.MOBILE,
    "Cls.model_type": ModelType.SERVER,
    ......
    }
  )
  ```

  - 三个模型是可以自由切换（MOBILE or SERVER）组合的（除了v4 cls没有SERVER版本），需要支持分别选择后一起下载，并且支持切换
  - 其他如CUDA、阈值、识别批处理大小等都删了不需要手动设置（前端也要删掉对应内容）

- （已手动修改）Result解析：Fixed. The issue was that RapidOCR() now returns a RapidOCROutput dataclass instead of a tuple. Changed process_image to use the object's .boxes, .txts, and .scores attributes directly.
